# **Library Management System – Project Statement**

## ** Problem Statement**
Managing library records manually becomes time‑consuming and prone to errors. Tasks such as adding books, issuing books, returning books, and tracking borrowed items often lead to confusion when handled on paper. There is a need for a simple, reliable, and beginner‑friendly software solution that automates these operations and stores data safely.

---

## ** Scope of the Project**
- Development of a console‑based library management application.
- Management of books, members, and issued book records.
- Storage and retrieval of data using JSON files.
- Providing basic administrative functionalities such as viewing issued books.
- Ensuring data remains persistent even after the program is closed.

---

## ** Target Users**
- School and college libraries that need a basic digital system.
- Small community libraries.
- Students learning file handling and Python programming.
- Beginners who want to understand how to build a real‑world project.

---

## ** High-Level Features**
- Add new books with ID, title, author, and number of copies.
- View all available books along with availability status.
- Issue books to members with automatic tracking.
- Return books and update inventory.
- Admin‑only dashboard to view all issued books.
- JSON‑based data storage for persistence.
- Error handling for invalid inputs and duplicates.
